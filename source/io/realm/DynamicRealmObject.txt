^^^^^^^^^^^^^^^^^^^^^^^^

.. _io_realm_DynamicRealmObject:



io.realm
========


 | `java.lang.Object <https://docs.oracle.com/javase/7/docs/api/java/lang/Object.html>`__ 
 | 		:ref:`io.realm.RealmObject <io_realm_RealmObject>` 
 | 					io.realm.DynamicRealmObject


**Implemented interfaces:**



- **io.realm.internal.RealmObjectProxy (?)**
- :ref:`io.realm.RealmModel <io_realm_RealmModel>` 


Class that wraps a normal RealmObject in order to allow dynamic access instead of a typed interface. Using a DynamicRealmObject is slower than using the regular RealmObject class.

hashCode

   :header-rows: 1
   
   * -  |   public **int** hashCode ()
   
   * - Returns a hash code value for the :ref:`DynamicRealmObject <io_realm_DynamicRealmObject>`  object.
       
       By the general contract of `Object.hashCode() <https://docs.oracle.com/javase/7/docs/api/java/lang/Object/hashCode().html>`__ , any two objects for which **equals (?)** returns ``true``  must return the same hash code value.
       
       
       
       Note that a :ref:`RealmObject <io_realm_RealmObject>`  is a live object, and it might be updated by changes from other threads. This means that a hash code value of the object is not stable, and the value should be neither used as a key in HashMap nor saved in HashSet.
       
       
       
       
       
       
       
       
       
       
       
       
       
       
       **Overrides**
       
       ``hashCode`` in class `Object <https://docs.oracle.com/javase/7/docs/api/java/lang/Object.html>`__

   * - Sets the reference to a :ref:`RealmSet <io_realm_RealmSet>`  on the given field.
       
       This will copy all the elements in the set into Realm, but any further changes to the set will not be reflected in the Realm.        - ``set`` - set of objects. Must either be primitive types or :ref:`DynamicRealmObject <io_realm_DynamicRealmObject>` s.

       - `IllegalArgumentException <https://docs.oracle.com/javase/7/docs/api/java/lang/IllegalArgumentException.html>`__  - if field name doesn't exist, it is not a set field, the objects in the set doesn't match the expected type or any Realm object in the set belongs to a different Realm.
