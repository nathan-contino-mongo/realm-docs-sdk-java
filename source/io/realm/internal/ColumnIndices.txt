io.realm.internal
=================

`java.lang.Object <https://docs.oracle.com/javase/7/docs/api/java/lang/Object.html>`__ 

Utility class used to cache the mapping between object field names and their column indices. The ``ColumnIndices``  instance is dedicated to a single **io.realm.BaseRealm (?)** instance. Different Realm instances will never share the same column indices cache. The column info cache is loaded lazily. A :ref:`ColumnInfo <io_realm_internal_ColumnInfo>`  will be added to the cache when the relevant Realm object gets accessed.

This class can be mutated, after construction, in two ways:



- the ``copyFrom``  method
   
   
   
   
- mutating one of the ColumnInfo object to which this instance holds a reference
   
   


Immutable instances of this class protect against the first possibility by throwing on calls to ``copyFrom`` . :ref:`ColumnInfo <io_realm_internal_ColumnInfo>`  for its mutability contract. There are two, redundant, lookup methods, for schema members: by Class and by String. Query lookups must be done by the name of the class and use the String-keyed lookup table. Although it would be possible to use the same table to look up ColumnInfo by Class, the class lookup is very fast and on a hot path, so we maintain the redundant table.
