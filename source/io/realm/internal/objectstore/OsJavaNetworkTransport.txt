.. list-table::
   :header-rows: 1
   
   * -  |   public abstract **void** sendRequestAsync (
        |      `String <https://docs.oracle.com/javase/7/docs/api/java/lang/String.html>`__  method, 
        |      `String <https://docs.oracle.com/javase/7/docs/api/java/lang/String.html>`__  url, 
        |      **long** timeoutMs, 
        |      `java.util.Map\<java.lang.String, java.lang.String> <https://docs.oracle.com/javase/7/docs/api/java/util/Map.html>`__  headers, 
        |      `String <https://docs.oracle.com/javase/7/docs/api/java/lang/String.html>`__  body, 
        |      **long** completionBlockPtr
        |   )
   
   * - This method is being called from JNI in order to execute the network transport itself. All logic around retry and parsing of results should be done by ObjectStore. Warning: This method is not allowed to throw. Any exception should be wrapped in a :ref:`Response <io_realm_internal_objectstore_OsJavaNetworkTransport_Response>`  and be returned as a result. This method assumes the network request is executed in an asynchronous manner. The actual request itself should always be executed within :ref:`executeRequest(String, String, long, Map, String) <io_realm_internal_objectstore_OsJavaNetworkTransport_executeRequest_String__String__long__Map__String_>`  as that allow sub-classes to modify the behaviour on the thread actually doing the request.
       
       
       
       
       
       
       **Parameters**
       
       - ``method`` - Which kind of HTTP method in lowercase.
          
          
          
       - ``url`` - Url to connect to.
          
          
          
       - ``timeoutMs`` - How long does the request has to complete?
          
          
          
       - ``headers`` - Which headers to send?
          
          
          
       - ``body`` - Which body to include?
          
          
          
       - ``completionBlockPtr`` - completion block that must be called on the C++ side. Must be passed through :ref:`handleResponse(Response, long) <io_realm_internal_objectstore_OsJavaNetworkTransport_handleResponse_Response__long_>` .
