Class PendingRow
^^^^^^^^^^^^^^^^

.. _io_realm_internal_PendingRow:



io.realm.internal
=================

`java.lang.Object <https://docs.oracle.com/javase/7/docs/api/java/lang/Object.html>`__ 

*Implemented interfaces:*



- :ref:`io.realm.internal.Row <io_realm_internal_Row>` 


A PendingRow is a row relies on a pending async query. Before the query returns, calling any accessors will immediately throw. In this case run :ref:`executeQuery() <io_realm_internal_PendingRow_executeQuery__>`  to get the queried row immediately. If the query results is empty, an :ref:`InvalidRow <io_realm_internal_InvalidRow>`  will be returned. After the query returns, :ref:`FrontEnd.onQueryFinished(Row) <io_realm_internal_PendingRow_FrontEnd_onQueryFinished_Row_>`  will be called to give the front end a chance to reset the row. If the async query returns an empty result, the query will be executed again later until a valid row is contained by the query results.
